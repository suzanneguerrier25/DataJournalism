Bird Calls: Injured Birds in NYC 2022

Context:

I'm using data from NYC's Urban Park Rangers (who are called by NYC residents to attend to animals, for example, a stray cat, a racoon, or in this case, birds). This dataset contains the status of the animals (healthy, unhealthy, injured), species (ex. hawk), and location (ex. Prospect Park), as well as how the animal was attended to. I'm using this data to analyze where birds have been injured in NYC, to raise awareness about the Park Ranger Service. By seeing the data for injured birds, NYC residents (or visitors) will hopefully be more motivated to make a call themselves. Personally, I had never heard of this service beforehand, despite being an NYC resident.

Data Sources:

Urban Park Ranger Animal Condition Response

Provided by the Department of Parks and Recreation (DPR), Accessed through NYC OpenData

Data Cleansing Pipeline:

  1. Downloaded the data as a csv file and import to google sheets
  2. Used google sheets to edit the data, deleting columns that weren't necessary to my narrative (ex. if the police was called) and removing rows that contained non-birds / non-raptors (eg. hawk)
  3. Downloaded the edited csv file to VS Code, and used a format_data.py file in conjunction with the json library to make a json file with the organized data
  4. Organized the data as a dictionary with the time and date as the key- separating the indivdual calls, and a list with the other attributes as the values
  5. I also removed any extra space as well as edited the Birds, Domestic to be Domestic Birds to facilitate conversion. I also edited any addresses with quotations for the same purpose

Prototyping and Designing:

Main Page

My initial idea for the main page was to have a map of NYC, with data for bird calls in each borough, and a summary on hovering a borough. I wanted to keep this summary simple by expanding to simply number of injured birds per borough, which conveys my message by showing how the data is spread across boroughs.

Borough Specific

For my borough specific pages, I planned to have a map of each borough, highlighting the parks, since that is where most birds were called in. I also added multiple possible charts summarizing the data. Firstly, number of injured birds by location, which I felt conveyed my message because when looking at the data per location, inhabitants of each location can know to look out for birds. Furthermore, I felt a bar chart would most effectively communicate the information.

Alternatively I considered a breakdown by species of bird, which I felt would convey my message by pinpointing which type of bird to look out for, as some might get more injured than others. I wanted to keep the graph simple with a bar chart, which would visually separate each species. I would also highlight the most injury-prone species with a darker color.

Finally, I considered a line graph on what times of year calls were being made, as that would pinpoint a time where residents should be the most vigilant. I chose a line graph as it would most simply show trends in certain times of the year.

I also included a potential color scheme, limiting my palette to a few colors to highlight key information.


Feedback

Peer feedback I received can be broken down into 2 main points
- Simplicity: The excess of graphs could distract the reader from the message, especially if unorganized
- Message: Be more clear with how the graphs help the reader to draw conclusions
To address these, I plan to clearly state on the specific pages how each graph relates to my message, and also potentially crowdsourcing data for if my peers knew about the park ranger call-in beforehand. Furthermore, I will separate the species graph (and potentially others) onto an additional information page, if the visitor would like to know more. On this page I will also post relevant resources.
